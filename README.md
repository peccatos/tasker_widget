# System Resource Widget

Данный репозиторий содержит два связанных, но технически независимых проекта:
1. **Web-Preview (Симулятор)** — браузерная версия на React/Tailwind для быстрой демонстрации UI/UX.
2. **Native C++ App** — исходный код реального десктопного приложения на Qt6 для Windows.

---

## 📁 Структура проекта

При скачивании проекта вы увидите следующее разделение:

*   `/src/`, `/index.html`, `package.json` и др. — **Web-Preview**. Это код для браузерной симуляции (React + Vite).
*   `/native-cpp/` — **Исходный код нативного C++ приложения**. Именно эту папку нужно открывать в Qt Creator или CLion для компиляции `.exe` файла.

---

## 🖥️ Native C++ App (Папка `/native-cpp/`)

Это ядро проекта. Виджет работает напрямую с Windows API для получения метрик CPU и RAM, а также управляет системными процессами.

### ⚙️ Конфигурация VPN (`config.json`)
Для того чтобы кнопка VPN открывала ваш кастомный клиент (а не стандартные настройки Windows), создайте файл `config.json` рядом с исполняемым файлом `.exe` со следующим содержимым:
```json
{
  "vpn_path": "C:\\Program Files\\MyCustomVPN\\vpn.exe",
  "force_hide_gpu": false
}
```
Если файл отсутствует, по умолчанию откроются системные настройки VPN Windows 10/11.

### 🎮 Интеграция GPU (NVAPI / AMD ADL)
Для получения реальных метрик видеокарты (Температура, Загрузка, VRAM) в C++ код уже полностью интегрирована поддержка **NVIDIA NVAPI** и **AMD ADL**. 
Из-за лицензионных ограничений сами SDK не включены в репозиторий, их нужно скачать и положить в папку проекта перед сборкой.

#### 🔗 Ссылки на скачивание SDK:
*   **NVIDIA (NVAPI):** [Скачать NVIDIA Display API (NVAPI) SDK](https://developer.nvidia.com/nvapi)
*   **AMD (ADL):** [Скачать AMD Display Library (ADL) SDK](https://gpuopen.com/advanced-display-library-adl/)

#### 🛠️ Инструкция по установке SDK:
1. **Для NVIDIA:** Скачайте NVAPI, создайте папку `native-cpp/NVAPI/` и положите туда содержимое архива (чтобы путь `native-cpp/NVAPI/nvapi.h` был доступен).
2. **Для AMD:** Скачайте ADL, создайте папку `native-cpp/ADL/` и положите туда папку `include` из архива (чтобы путь `native-cpp/ADL/include/adl_sdk.h` был доступен).
3. Откройте `native-cpp/CMakeLists.txt` и раскомментируйте строки `include_directories` и `target_link_libraries` (для NVAPI).
4. Откройте `native-cpp/GpuMonitor.cpp` и раскомментируйте `#define ENABLE_NVAPI` или `#define ENABLE_AMD_ADL` в самом начале файла.
5. Скомпилируйте проект.

*Примечание: Если на устройстве пользователя нет дискретной видеокарты (например, офисный ноутбук или Mac), виджет автоматически адаптируется и скроет панель GPU/VRAM, оставив только CPU и ОЗУ.*

## 🌐 Web-Preview (Браузерная версия)

Текущая веб-версия — это **интерактивный симулятор**. 
Она создана для того, чтобы новые разработчики, менеджеры или пользователи могли быстро понять логику работы приложения без необходимости скачивать исходный код, настраивать CMake, компилятор C++ и собирать `.exe` файл.

### Особенности Web-Preview:
- **Симуляция данных**: Метрики генерируются динамически.
- **Индикация процессов**: При нажатии на кнопки появляются всплывающие уведомления (Toasts), объясняющие вызовы системных процессов.
- **Адаптивность под устройства**: В превью добавлена специальная панель управления, где можно симулировать запуск виджета на устройстве **без дискретной видеокарты** (GPU скрывается).
- **Расширенный фрейм**: Контейнер VRAM расширен, чтобы комфортно вмещать большие объемы памяти (например, `16.50/32.00 GB`).
